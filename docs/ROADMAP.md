# 実装計画（概要）

> ヒント程度のメモ。詳細は着手時に再考する前提。フェーズ順は前後してよい。

## v1（現状のスキャフォルド）

- コアループ: カタログ → ランダムカード → タップでマーク → ビンゴ判定 → 達成演出
- PWAシェル（manifest + Service Worker、`vite-plugin-pwa`）
- 永続化は `localStorage`（`src/storage.ts`）。散歩の中断・再開に対応
- 読み上げ（Web Speech API、字が読めない幼児向けの補助）

## v1.x — 写真 / 思い出アルバム

- **不変条件: 写真はサーバに送らない。端末ローカル(IndexedDB)のみ。** これにより公開版でもプライバシー責任を負わない。
- 撮影: マス長押し → `<input type="file" accept="image/*" capture="environment">`
- 保存: `idb` で blob を IndexedDB に。`src/storage.ts` のメモ位置に API を足す
- アルバム画面: IndexedDB を読むだけ。退避は `navigator.share({ files })`
- 検討: ストレージ退避(eviction)対策、容量上限の扱い

## v2 — 絵の差し替え / テーマ

- 手描きSVGアイコン: `CatalogItem.asset` を有効化し、描画を emoji → `<Icon id>` に切替。**id は不変**なので既存カード・保存データは無改修
- テーマ別カードパック（公園版・駅前版など）: カタログに `pack` タグ、設定でパック選択
- 検討: 季節・地域バリエーション、難易度（お題の出現頻度）

## v3 — 公開（必要になったら）

- **写真ローカル固定・アカウント無しを維持すれば、配るのは静的 `dist/` だけ＝バックエンド不要**
- ホスティング: GitLab Pages / Cloudflare Pages
- 検討: i18n、効果音（WebAudio）、簡単な利用説明、OGP

## 横断的に効いてくる項目（ヒント）

- アクセシビリティ: 屋外・直射日光下のコントラスト、タップ領域の大きさ
- 効果音: マーク音 / ビンゴ音（読み上げと排他制御）
- 誤操作対策: 操作系（設定・新カード）はヘッダ／シートに隔離済み。必要なら確認ダイアログ
- テスト: `bingo.ts`（判定）と `random.ts` は Vitest で単体テストしやすい純関数
- 計測: 必要なら自前の軽量イベントのみ（外部送信は方針に反するため慎重に）
